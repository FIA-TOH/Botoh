# Regras do projeto

## Internacionalização
- Todo texto visível ao usuário deve usar o sistema de tradução existente.
- Sempre que adicionar ou alterar texto na interface, criar ou atualizar também as chaves em `pt`, `en` e `es`.
- Sempre que adicionar ou alterar mensagens retornadas pelo backend ao usuário, registrar também as traduções correspondentes no i18n do backend.
- Não deixar strings hardcoded em português na UI, exceto conteúdo técnico que não seja exibido ao usuário.
- Antes de finalizar mudanças com texto novo, verificar se há strings visíveis fora do sistema de tradução.
